Website for Conference

Back to Home

This child website was one of the first sites I have made. Its purpose was to advertise and organize the annual IPATA conference in 2016. It could be found at www.ipata.org/kul-conference-home. It was designed on a website that already existed.

The site needed to accommodate all the different aspects of the conference and organize them for easy access. It also had to be mobile friendly. The home page displays each of the other pages with a little blurb.

It was made using HTML and CSS along with a website editing software owned by the company. Web pages included: Home, Ads & Exhibits, Attendees, Awards, FaceTime, Hotel, Registration, Schedule, Silent Auction, Sponsorship, Things to Do, Travel, Visa Letter.

One of the challenges I faced was making sure to create a mobile friendly site. By using Media Queries and a specific mobile image, I was able to create a design that kept the theme alive.

Examples of a few of the other pages are shown below. Most pages were information specific. The right sidebar was the same on each page using a widget.

The Process

This website was made using a content manager provided by the company and with handwritten HTML and CSS. Any CSS had to be written out and much of the HTML was hand written as well.

To begin, research was done to see what ideas were already out there for conferences. Then I considered the purpose and needs of the website.

Designs were drawn out by hand and then replicated in Photoshop. At first, the ideas that I had were rough and would not have worked in the space allowed. But I soon realized my error and created designs that would work along with ideas about the mobile version of these pages.

After realizing some limitations on the site, I recreated my original ideas in Photoshop. The entire process was monitored by the Senior Web Developer.

While the front page was being discussed, I also created corresponding mock ups for some of the other pages that would be created.

Once a design was settled on, I went ahead and created the rest of the site, verifying with my team for the information and for their approval.

This process took about 5 months to make.
